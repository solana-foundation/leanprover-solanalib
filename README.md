# Solanalib

[![CI](https://github.com/solana-foundation/leanprover-solanalib/actions/workflows/ci.yml/badge.svg)](https://github.com/solana-foundation/leanprover-solanalib/actions/workflows/ci.yml)
[![Docs](https://github.com/solana-foundation/leanprover-solanalib/actions/workflows/docs.yml/badge.svg)](https://solana-foundation.github.io/leanprover-solanalib/)

A library of formal models and verified results for Solana programs, written
in [Lean 4](https://lean-lang.org/). Solanalib is to Solana what
[Mathlib](https://leanprover-community.github.io/mathlib4_docs/) is to
mathematics: a shared, well-documented foundation that downstream
verification work — on [Anchor](https://www.anchor-lang.com/),
[Pinocchio](https://github.com/anza-xyz/pinocchio), or hand-rolled
programs — can build on without redefining the same primitives every time.

**Documentation:** <https://solana-foundation.github.io/leanprover-solanalib/>

## Status

Early scaffold. The first theorem (`Solanalib.Account.transfer_preserves_total`)
proves that lamport transfers conserve total balance — the foundation every
richer account invariant eventually reduces to.

## Layout

```
Solanalib.lean                            -- root module
Solanalib/
├── Primitives/
│   └── Lamports.lean                     -- the native value unit
└── Account/
    ├── Basic.lean                        -- Account, credit, debit
    └── Transfer.lean                     -- transfer + conservation theorem
```

Modules are organised by Solana concept (Account, Token, Pda, …), not by
framework. Framework-specific layers (Anchor account validation, Pinocchio
entrypoint patterns) will live in their own subdirectories on top of the
core primitives.

## Prerequisites

Lean is managed by [`elan`](https://github.com/leanprover/elan), Lean's
toolchain installer (the analogue of `rustup`). Install it once:

```sh
curl -sSf https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh | sh
```

The pinned toolchain version lives in `lean-toolchain`; `elan` reads it
automatically.

[`just`](https://github.com/casey/just) is used as a command runner:

```sh
brew install just
```

## Build

First-time setup pulls dependencies (Mathlib, doc-gen4) and downloads
Mathlib's pre-built oleans so you don't compile it from scratch:

```sh
just setup
```

Type-check the library:

```sh
just build
```

## Documentation

The documentation site is generated by
[`doc-gen4`](https://github.com/leanprover/doc-gen4), the same tool that
produces the [Mathlib docs](https://leanprover-community.github.io/mathlib4_docs/).
The output style — sidebar navigation, hyperlinked signatures, rendered
docstrings — comes from there.

```sh
just docs        # build to .lake/build/doc/
just docs-open   # build and open in the default browser
```
